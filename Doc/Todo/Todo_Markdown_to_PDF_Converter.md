# Markdown to PDF Converter

## Overview
Создание системы для генерации PDF из структурированной Markdown документации с кастомизацией через HTML шаблоны. Система должна поддерживать:
- Ссылки на другие MD файлы (рекурсивное объединение)
- Изображения
- Разные HTML шаблоны для разных типов страниц
- Типографику: шрифты, колонтитулы, отступы, рамки страниц

## Step-by-step

1. Выбрать технологический стек (Node.js + Puppeteer)
2. Создать структуру проекта (директории templates, examples, output)
3. Создать package.json с зависимостями (markdown-it, puppeteer, gray-matter, glob)
4. Реализовать парсер Markdown файлов с разрешением ссылок на другие MD файлы
5. Создать систему шаблонов HTML для разных типов страниц
6. Реализовать движок рендеринга HTML из Markdown с применением шаблонов
7. Настроить CSS для типографики (шрифты, колонтитулы, отступы, рамки)
8. Интегрировать генератор PDF через Puppeteer
9. Создать CLI интерфейс для запуска конвертера
10. Протестировать на примерах с реальными MD файлами
11. Упаковать в standalone exe файл с помощью pkg
12. Написать документацию по использованию

## Progress

### Шаг 1: Выбор технологического стека ✓
**Решение:** Node.js + Puppeteer

**Обоснование:**
- Максимальная гибкость в HTML/CSS шаблонах
- Полная поддержка CSS Paged Media для колонтитулов
- Возможность использовать любые веб-шрифты
- Простая отладка через браузер
- Возможность упаковки в standalone exe файл (~100 МБ) без требований к окружению

**Основные библиотеки:**
- `markdown-it` - парсинг Markdown в HTML
- `puppeteer` - генерация PDF через headless Chrome
- `gray-matter` - парсинг frontmatter в MD файлах
- `glob` - поиск файлов по паттернам
- `pkg` - упаковка в exe (финальный этап)

**Архитектура:**
```
Markdown файлы → MD Parser → HTML Template Engine → PDF Generator → PDF файл
```
